# 模拟统计数据不匹配问题分析

## 问题现象
模拟完成后显示：
- ✅ 模拟成功完成！
- 📊 统计: 100个用户, 10个帖子, 15条评论  
- 总用时: 214.10秒

但实际数据库 `weibo_test\sim_20260203_121042.db` 中：
- 几乎都是 sign up 动作
- 只看得到一个帖子

## 根本原因

在 `visualization_system\backend\simulation\executor.py` 的第 **239-241** 行发现问题：

```python
self.task.stats.posts_created += 2      # 假数据！
self.task.stats.comments_created += 3   # 假数据！
await asyncio.sleep(0.5)
```

### 问题分析：

1. **硬编码的假统计数据**
   - 每一轮结束后，无论实际发生什么，都会固定累加：+2个帖子、+3条评论
   - 5轮模拟 → 显示10个帖子、15条评论（完全是假数据）

2. **实际模拟行为**（第221-237行）
   - 如果启用LLM且模型可用：调用所有Agent执行LLM动作
   - 如果未启用LLM或模型不可用：
     - 第一轮（round 0）：Agent 0创建一个测试帖子
     - 其他轮次：跳过，不做任何动作
   
3. **为何数据库只有sign up和一个帖子**
   - 100个用户的sign up是真实的（通过 `env.reset()` 创建）
   - 只在第一轮创建了一个测试帖子
   - 其他轮次因为没有启用LLM或LLM未正常工作，没有产生任何帖子/评论

## 解决方案

### 修复内容
删除硬编码的假统计，改为从数据库读取真实数据：

```python
# 8. 从数据库读取真实统计数据
self._log("📊 统计数据库中的真实数据...")
import sqlite3
try:
    conn = sqlite3.connect(str(self.task.db_path))
    cursor = conn.cursor()
    
    # 统计帖子数量
    cursor.execute("SELECT COUNT(*) FROM posts")
    self.task.stats.posts_created = cursor.fetchone()[0]
    
    # 统计评论数量
    cursor.execute("SELECT COUNT(*) FROM comments")
    self.task.stats.comments_created = cursor.fetchone()[0]
    
    conn.close()
except Exception as e:
    self._log(f"⚠️ 读取统计数据失败: {e}")
```

### 效果
现在模拟完成后，统计数据将显示真实的数据库内容：
- 如果LLM未启用：会显示1个帖子、0条评论（符合实际）
- 如果LLM正常工作：会显示LLM实际生成的帖子和评论数量
