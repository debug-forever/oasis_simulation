# 快速测试指南

## 🚀 测试数据库自动切换功能

### 前提条件

确保后端和前端服务都在运行：

1. **后端服务** (端口 8001)
   ```powershell
   cd E:\Project\oasis_simulation\visualization_system\backend
   uvicorn main:app --host 0.0.0.0 --port 8001 --reload
   ```

2. **前端服务** (端口 5173)
   ```powershell
   cd E:\Project\oasis_simulation\visualization_system\frontend
   npm run dev
   ```

---

## 📝 测试步骤

### 步骤 1: 启动模拟

1. 打开浏览器访问 http://localhost:5173
2. 导航到 **Simulation (模拟控制台)** 页面
3. 配置参数（建议使用以下配置快速测试）:
   - **Agent数量**: 5
   - **模拟轮次**: 2
   - **启用LLM**: ❌ (演示模式更快)
   - **启用初始帖子**: ✅
   - **初始帖子数量**: 2
4. 点击 **"🚀 启动模拟"** 按钮

### 步骤 2: 观察后端日志

在后端终端窗口，你应该看到类似输出：

```
[18:30:20] 🚀 开始模拟任务...
[18:30:20] 配置: 5个agent, 2轮
[18:30:21] 📦 导入Oasis模块...
[18:30:22] 🌱 播种初始帖子...
[18:30:23] ▶️ 开始执行2轮模拟...
...
[18:30:45] ✅ 模拟完成！数据库: weibo_test/weibo_sim_20240204_183045.db
[18:30:45] 总用时: 25.3秒
[18:30:45] 🔄 切换数据库连接到新生成的模拟数据库...    ⬅️ 关键！
[18:30:45] ✅ Database switched to: weibo_test/...         ⬅️ 关键！
[18:30:45] ✅ 数据库连接已切换！前端现在可以查看新的模拟结果  ⬅️ 关键！
```

> ✅ **关键验证点**: 看到 "数据库连接已切换" 消息

### 步骤 3: 前端通知验证

在前端页面，你应该看到：

1. ✅ **任务状态** 变为 "✅ 已完成"
2. ✅ **进度条** 显示为绿色（100%）
3. ✅ **右上角弹出通知**:
   ```
   ✅ 模拟完成！
   任务 xxx 已成功完成！
   用户: 5, 帖子: X, 评论: X
   ```
4. ✅ **"查看结果"** 按钮可点击

### 步骤 4: 查看结果

点击以下任一方式查看结果：

- **方式1**: 点击通知消息
- **方式2**: 点击任务列表中的 **"查看结果"** 按钮

**预期结果**:
- 自动跳转到 Dashboard 页面
- Dashboard 显示新模拟的数据

### 步骤 5: 验证数据

在 Dashboard 页面检查：

| 指标 | 预期值 | 说明 |
|------|--------|------|
| 总用户数 | 5 | 应等于配置的 Agent 数量 |
| 总帖子数 | > 2 | 至少有初始帖子 |
| 总评论数 | >= 0 | 可能为0（演示模式） |

> ❌ **失败标志**: 如果显示 "100 用户 0 帖子 0 评论"，说明切换未成功
> ✅ **成功标志**: 显示真实的模拟数据（5用户，>2帖子）

## 🐛 故障排除

### 问题1: 后端没有显示 "数据库连接已切换"

**可能原因**: 代码未更新

**解决方法**: 
1. 确保后端服务器重启（uvicorn reload）
2. 检查 `executor.py` 是否包含新代码

### 问题2: 前端没有弹出通知

**可能原因**: 前端轮询未工作

**解决方法**:
1. 检查浏览器控制台是否有错误
2. 刷新页面重试

### 问题3: Dashboard 仍显示旧数据

**可能原因**: 数据库切换失败

**解决方法**:
1. 检查后端日志中的错误信息
2. 运行 `python test_db_switch.py` 测试
3. 手动检查 `weibo_test/` 目录是否有新文件
